# Thriller Chapter Queue Prompt

Use the `thriller-forge-for-pi` skill.

## Goal

Create compact, draftable chapter packets for the active thriller book.

## Read first

- `series/series-bible.md`
- `series/reveal-ledger.md`
- `series/continuity-ledger.md`
- active book `book-bible.md`
- active book `plot-ladder.md`
- active book `threat-clock.md`
- active book `chapter-queue.md`

## Required output

Update only:

- active book `chapter-queue.md`
- `series/reveal-ledger.md`
- `series/continuity-ledger.md`
- `STATUS.md`

## Packet format

Each chapter packet should include:

- chapter number
- POV
- purpose
- pressure movement
- clue or reveal movement
- character pressure
- scene engine
- required continuity
- ending hook
- status

## Rules

Do not draft prose in this pass. Do not create extra artifacts. If the book architecture is not ready, add a revision ticket instead of inventing unsupported chapter turns.
