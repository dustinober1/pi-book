# Book Planning

Plan the active book's causal architecture, POV rules, character pressure, opposition, setup/payoff IDs, ending contract, research dependencies, and next-book handoff.

Create a compact `remarkability.yaml` contract that names the safe obvious version to avoid, the author-only advantage, productive discomfort, retellable hook, 2–5 signature moments, productive reader disagreements, restrained recurring motifs, lingering question, hand-sell reason, and accepted reader costs.
