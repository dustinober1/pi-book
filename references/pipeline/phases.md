# Book Genesis Universal Pipeline

## Phase 0: Intake

- receive the user's basic idea
- infer assumptions
- persist assumptions and brief
- initialize creative sovereignty: author intent, taste profile, risk budget, and discarded choices
- generate review personas that can detect false voice, generic polish, sentimentality, genre betrayal, and fake specificity early

## Phase 1: Foundation

- premise expansion
- theme
- characters
- subplot seeds and relationship pressures
- voice bible and anti-voice rules
- author voice fingerprint from samples or stated taste
- human source bank for permitted lived material
- web-backed name collision audit
- name and entity originality filter
- market framing

## Phase 2: Architecture

- outline
- scene-level therefore/but causality chain
- scene embodiment map for physical action, objects, task pressure, and embodied scene friction
- 2-3 integrated subplot arcs
- continuity ledger
- reader promise tracker
- drift and loop alarm
- emotional curve
- opening strategy
- ending promise

## Phase 3: Drafting

- write in chapter blocks
- weave and update subplot beats
- update continuity after every chapter block
- update reader promises and chapter scorecards
- update human specificity ledger with restraint
- update scene embodiment, discarded choices, subtext, read-aloud ear, and over-polish notes
- check drift and loop hard stops
- preserve structural variation
- update state after each block

## Phase 4: Adversarial Audit

- run structural criticism before score
- run StoryScope-informed narrative fingerprint audit for thematic over-determination, tidy single-track plots, and AI-shaped narrative choices
- run source-informed AI tell mitigation audit for visible prose, punctuation, formatting, and subtext tells
- run subtext, read-aloud ear, and over-polish audits to protect author fingerprint and reader trust
- pressure-test the manuscript against review personas so false notes and workshop-clean generic prose get caught before scoring
- run creative sovereignty, scene embodiment, negative capability, and revision philosophy passes
- convert findings into revision tickets
- cut, merge, reorder, or rewrite when needed

## Phase 5: Final Score

- run Genesis Score
- record gate verdict and next action
- write `artifacts/09-genesis-score.md`

## Phase 6: Editorial Package

- synopses
- cover brief
- formatting package
- positioning strategy
- reader response plan and beta feedback log
