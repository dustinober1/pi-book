# Recurring Formal Device Tracker

## Purpose

When a manuscript uses a distinctive structural device (machine-log chapters, epistolary sections, in-world documents, data dumps, dream sequences, flashbacks, alternate POV chapters, etc.), this artifact tracks each occurrence to ensure the device escalates rather than repeats.

The risk: a device that is powerful on first use becomes predictable and diminishing on subsequent uses if each occurrence does not do substantively new work.

## Device rules

- First use: full, immersive, establishes the device's power and tone. Keep mostly intact.
- Second use: must introduce something substantively new — a new horror, a new dimension of the system, a new kind of information, or a direct threat to a named character.
- Third and subsequent uses: compress aggressively. Preserve only the most revealing elements. Cut redundant metrics, repeated formatting, or information the reader already has.
- Escalation requirement: each occurrence must feel more compressed AND more chilling/revealing than the last. If it feels like the same file with a different name, cut or merge.
- Total occurrence budget: for a novel-length manuscript, most formal devices should not exceed 4-6 occurrences. Each occurrence beyond the budget must earn its place with new, irreplaceable work.

## Fatigue check

For each occurrence, answer:
1. Does this occurrence reveal something the reader could not have inferred from earlier ones?
2. Does this occurrence directly build dread, complicate the plot, or threaten a character we care about?
3. Would cutting this occurrence weaken the book?
If the answer to all three is no, the occurrence is redundant.
