## Canonical name and term lock

Once a name, location, institution, numerical value, or technical term is established, it is locked. Every subsequent use must match exactly or have an explicit in-story justification for variation.

| canonical form | type | first appearance | locked in chapter | alternate forms to catch | last verified |
| --- | --- | --- | --- | --- | --- |

Types: character name, location, institution, technical term, numerical value, world fact, object, date/time reference

Rules:
- when a new named entity first appears, add it to this lock table with the exact form used
- before each chapter block, search the manuscript for every locked term and verify consistency
- if an alternate form is found, flag it immediately as a continuity error and standardize
- for numerical values (horizons, specifications, dates, distances), lock the value and flag any divergence
- this is a structural integrity check, not a copy-editing pass; resolve before final scoring

## Infrastructure and world-building seeding

Track when institutional, legal, political, or systemic frameworks are introduced. For speculative fiction and thrillers, these must be visible before they need to be functional.

| framework | type | first seeded | made functional | adequate seeding? |
| --- | --- | --- | --- | --- |

Types: legal authority, political structure, institutional history, technical specification, operational protocol, jurisdictional scope

Rules:
- if a framework is relied on for a major plot beat, it must be introduced at least two chapters before that beat
- "one layer at a time" explanations (e.g., Holt's "one streetlight at a time") should be seeded early and built on gradually
- if a reader would need to understand the framework for a scene to land, check whether the framework was introduced before the scene, not during it
