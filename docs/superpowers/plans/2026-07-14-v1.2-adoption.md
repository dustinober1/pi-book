# Novel Forge v1.2 Existing-Project Adoption Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Preview, remap, and atomically adopt DOCX, EPUB, Markdown, text, or chapter-directory manuscripts while preserving meaningful formatting and embedded images without changing the source.

**Architecture:** Treat source discovery and conversion as ephemeral, read-only work. Conversion engines produce one normalized `AdoptionPreview`; mapping operations transform that preview without touching the project; confirmation produces typed text and binary changes applied through one guarded transaction.

**Tech Stack:** TypeScript, Mammoth, fflate, fast-xml-parser, Turndown with GFM plugin, Pandoc when available, Node fs/crypto/child_process, TypeBox/YAML, existing transaction and wizard registry.

## Global Constraints

- Complete the wizard-foundation plan first.
- Never modify or delete the source manuscript.
- Refuse adoption when the active manuscript already contains chapter files.
- Reject encrypted, corrupt, empty, oversized, traversal-bearing, symlink-bearing, remote-resource, or unsupported archives before project mutation.
- Preserve meaningful structure, images, captions, alt text, links, notes, tables, footnotes, and scene breaks when safely representable.
- Normalize unsupported page layout rather than creating fragile Markdown.
- Mark discovered publishing metadata unverified until the author accepts it.
- Apply chapters, front/back matter, assets, metadata candidates, reports, counts, guidance, and Git checkpoint atomically.
- Invent no plot, canon, approval, review, reader evidence, or publishing value.

- [ ] **Step 4: Implement source resolution and safe archive inspection**

Canonicalize every entry with POSIX separators and reject if:

```ts
const normalized = posix.normalize(name.replace(/\\/g, "/"));

```

Reject `file:`, `http:`, and `https:` relationships during XML relationship scanning. Reject XML text containing `<!DOCTYPE` or `<!ENTITY` before parsing. Do not follow symlinks in directory adoption; use `lstatSync` and accept regular `.md`/`.txt` files only.
