# Novel Forge v1.2 Existing-Project Adoption Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Preview, remap, and atomically adopt DOCX, EPUB, Markdown, text, or chapter-directory manuscripts while preserving meaningful formatting and embedded images without changing the source.

**Architecture:** Source discovery and conversion are ephemeral and read-only. Conversion engines produce one normalized `AdoptionPreview`; mapping operations transform that preview without touching the project; confirmation produces typed text and binary changes applied through one guarded transaction.

**Tech Stack:** TypeScript, Mammoth, fflate, fast-xml-parser, Turndown with GFM plugin, Pandoc when available, Node fs/crypto/child_process, TypeBox/YAML, existing transaction and wizard registry.

## Global Constraints

- Complete the wizard-foundation plan first.
- Never modify or delete the source manuscript.
- Refuse adoption when the active manuscript already contains chapter files.
- Reject encrypted, corrupt, empty, oversized, traversal-bearing, symlink-bearing, remote-resource, or unsupported archives before project mutation.
- Preserve meaningful structure, images, captions, alt text, links, notes, tables, footnotes, and scene breaks when safely representable.
- Normalize unsupported page layout rather than creating fragile Markdown.
- Mark discovered publishing metadata unverified until the author accepts it.
- Apply chapters, front/back matter, assets, metadata candidates, reports, counts, guidance, and Git checkpoint atomically.
- Invent no plot, canon, approval, review, reader evidence, or publishing value.

```ts
export type AdoptionSectionKind =
  | "front-matter"
  | "chapter"

  markdown: string;

export interface AdoptionAsset {
  id: string;
  originalName: string;
  mediaType: string;
  bytes: Uint8Array;
  hash: string;
  width: number | null;

export interface AdoptionPreview {

  warnings: AdoptionWarning[];
  sourceWordCount: number;
  proposedWordCount: number;
}
```

Use opaque random IDs only in ephemeral state. Persistent files receive deterministic IDs and hashes during apply.

- [ ] **Step 4: Implement safe source resolution**

Canonicalize ZIP entries and reject unsafe names:

```ts
const normalized = posix.normalize(name.replace(/\\/g, "/"));
if (
  normalized.startsWith("/") ||
  normalized === ".." ||
  normalized.startsWith("../")
) {
  throw new Error(`Unsafe archive path: ${name}`);
}
```

Reject `file:`, `http:`, and `https:` relationships. Reject XML containing `<!DOCTYPE` or `<!ENTITY` before parsing. For directory adoption, use `lstatSync`, reject symlinks, and accept regular `.md`/`.txt` files only.
