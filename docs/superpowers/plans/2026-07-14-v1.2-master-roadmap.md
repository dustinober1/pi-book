# Novel Forge v1.2 Master Implementation Roadmap

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this roadmap plan-by-plan. Each linked plan uses checkbox (`- [ ]`) syntax for task tracking.

**Goal:** Deliver Novel Forge 1.2.0 with a secure temporary browser wizard, high-fidelity existing-project adoption, isolated reader experiments, complete publishing and marketing packaging, and context-aware next-book creation.

**Architecture:** The browser is a loopback-only read/propose surface. All accepted actions return to typed Novel Forge application services and the existing guarded transaction, Git checkpoint, undo, `STATUS.md`, and `HANDOFF.md` pipeline. Pandoc is preferred at runtime, while Node conversion paths keep every required workflow usable without it.

**Tech Stack:** TypeScript 5.9, Node.js 22.19+/24, Pi Coding Agent 0.80.3, TypeBox, YAML, Node HTTP/crypto/fs/child_process, static browser HTML/CSS/JavaScript, Mammoth, fflate, fast-xml-parser, Turndown, Marked, docx, ExcelJS, Busboy, open, Node test runner, GitHub Actions.

## Global Constraints

- Version target is exactly `1.2.0`.
- Work only on `agent/v1.2-existing-project-reader-usability`; open a PR against `main` and leave it unmerged.
- The browser never writes project files directly and never invokes arbitrary commands.
- Bind the wizard only to `127.0.0.1`; require one random session credential on every API request; reject unexpected origins; expire idle sessions.
- Every accepted mutation uses expected-stage and project-state-hash validation, atomic rollback, one Git checkpoint, and regenerated `STATUS.md` and `HANDOFF.md`.
- Preserve v1.1 projects without mandatory migration and preserve recorded reader evidence exactly during migration.
- Preserve source manuscripts read-only; never overwrite occupied manuscript destinations.
- Only accepted `source: human` rows count as reader evidence.
- Never invent publishing metadata, reader results, inherited canon, plot solutions, or marketing approval.
- Use Pandoc through argument arrays with fixed timeouts and controlled directories; never shell-interpolate untrusted values.
- Do not create `v1.2.0` until the merged `main` commit passes the complete Node 22.19.0 and Node 24 matrix.
