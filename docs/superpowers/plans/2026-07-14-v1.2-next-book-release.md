# Novel Forge v1.2 Next-Book and Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Create the next book from reviewed series context and explicit author decisions, record inheritance provenance without inventing canon, then complete 1.2 documentation, migration, verification, and PR preparation.

**Architecture:** A pure inheritance collector reads locked canon, open threads, series arc, prior book state, and bounded reader summaries into a proposal. The browser lets the author accept, defer, or exclude context and supply the new book contract. One refactored `addBook` application service writes the new book, inherited context, report, series arc, project state, guidance, and Git checkpoint atomically.

**Tech Stack:** TypeScript, TypeBox/YAML, existing canon/story-thread/series schemas, reader store, wizard registry, transaction/Git guidance, Node test runner and GitHub Actions.

## Global Constraints

- Complete the foundation and reader plans first; packaging must be complete enough to recognize canon-locked or packaged eligibility.
- Normal next-book creation requires the current book to be canon-locked or packaged.
- Forced creation remains Advanced, explicitly confirmed, and marked in provenance.
- Inherit only accepted existing facts, relationships, open threads, series-arc carries, and bounded reader findings.
- Never invent a plot solution, character outcome, new canon fact, reader result, title, protagonist, or series role.
- Author decisions must explicitly classify continuing, deferred, immutable, optional, and excluded context.
- Create the next book, series-arc update, inheritance files, project state, status, handoff, and Git checkpoint atomically.
- Leave the implementation PR unmerged and do not create `v1.2.0` until merged `main` passes the full matrix.
