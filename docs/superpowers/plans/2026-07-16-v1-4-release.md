# Novel Forge 1.4 — Author Velocity Release Plan

## Goal

Qualify, version, and release the complete Author Velocity train after all seven feature PRs are merged.

## Release contract

- Bump package, lockfile, installed-version constant, and new-project metadata to `1.4.0`.
- Preserve readability of 1.3 and earlier projects through advisory metadata upgrade.
- Verify brief bootstrap, premise selection boundary, persistent resume, rolling packet windows, structured rejections, and target autopilot together.
- Add clean-install and packed-extension coverage for the new commands and services.
- Replace the fixed 1.3 release-tree command with a current 1.4 verifier while retaining the historical 1.3 verifier and fixtures.
- Add release notes and a main-branch workflow that creates annotated tag and GitHub release `v1.4.0` only after Node 22.19.0 and Node 24 pass the exact merged commit.
- Do not claim wall-clock speed, automatic literary quality, automatic approval, or human reader validation.

## Verification

The final PR head and resulting main merge commit must pass:

```bash
npm ci
npm run typecheck
npm test
npm run eval
npm run verify:release
npm pack --dry-run
```

No temporary workflow, helper script, diagnostics, generated project, copied brief, package tarball, or release marker may remain in the merged tree.
