# Novel Forge v1.2 Wizard Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Establish the 1.2 compatibility contract, binary-safe atomic writes, canonical workflow schemas, and a secure temporary browser wizard that can inspect and propose actions without bypassing Novel Forge.

**Architecture:** Extend the schema and transaction layers first, then add a loopback-only Node HTTP server with a static browser client. A closed application registry supplies sanitized snapshots, previews, and guarded apply handlers; the browser never receives an arbitrary filesystem or command interface.

**Tech Stack:** TypeScript 5.9, Node.js 22.19+/24, TypeBox, YAML, Node HTTP/crypto/fs/child_process, Busboy, open, static HTML/CSS/JavaScript, Node test runner.

## Global Constraints

- Set package and project version to exactly `1.2.0`.
- Keep v1.1 projects readable before migration.
- Browser API requests require one session credential in `Authorization: Bearer <token>` and the exact loopback origin.
- Bind only to `127.0.0.1` and use an ephemeral port.
- Browser routes are read/propose only; accepted writes call typed application services.
- All accepted mutations retain expected-stage, project-hash, rollback, Git checkpoint, `STATUS.md`, and `HANDOFF.md` behavior.
- Binary files participate in the same transaction and rollback semantics as text files.
- Work only on `agent/v1.2-existing-project-reader-usability` and leave the eventual PR unmerged.

- [ ] **Step 5: Implement the static client shell**

Use a sidebar for four workflows and one main review panel. Dynamic text uses `textContent`, not `innerHTML`.

```js
const token = new URL(location.href).searchParams.get("token");
if (!token) throw new Error("Wizard session token is missing.");
history.replaceState({}, "", "/");
async function api(path, options = {}) {
  const headers = new Headers(options.headers || {});
  headers.set("Authorization", `Bearer ${token}`);
  if (options.body && !(options.body instanceof FormData)) headers.set("Content-Type", "application/json");
  const response = await fetch(path, { ...options, headers });
  const body = await response.json();
  if (!response.ok) throw new Error(body.message);
  return body;
}
```
