# Novel Forge v1.2 Packaging and Publishing Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn packaging into a guided repair wizard backed by canonical publishing/marketing metadata and reproducible DOCX, EPUB, Markdown, CSV, XLSX, audiobook, retailer, launch, social, advertising, and series-page outputs.

**Architecture:** Canonical YAML and accepted manuscript sources feed a deterministic package model. The checklist exposes typed repair actions. Export engines build and validate all required files in a temporary staging directory, then one guarded binary/text transaction publishes a new source-hashed export set and package manifest without overwriting approved prior outputs.

**Tech Stack:** TypeScript, TypeBox/YAML, Pandoc when available, docx, Marked, fflate, ExcelJS, Node crypto/fs/child_process, existing reader/adoption stores, wizard registry, transaction and Git guidance pipeline.

## Global Constraints

- Complete the foundation, adoption, and reader plans first.
- `publishing.yaml` and `marketing.yaml` are canonical; generated documents never become source of truth.
- Unknown metadata remains blank and visible; never invent ISBN, copyright, dates, categories, accessibility status, narrator, duration, or retailer facts.
- Generated marketing content is `draft` until the writer explicitly approves its asset group.
- Required manuscript/metadata conversion failure blocks final package completion.
- Marketing generation failure cannot corrupt manuscript exports or approved metadata.
- Produce exports in a session-owned staging directory and publish only after validation.
- Never silently overwrite approved outputs; changed source hashes create a new export set and mark old sets stale.
- Every accepted metadata, marketing, and package mutation uses guarded transactions, Git checkpoints, `STATUS.md`, and `HANDOFF.md`.
