# Novel Forge v1.6.0 — Deterministic Prose Lint

Novel Forge v1.6.0 adds a local, deterministic prose-lint layer for focused manuscript review.

## Added

- `npm run audit:prose -- <project-root>` produces read-only Markdown or JSON findings for mechanical mistakes, project consistency, repetition, and style-pattern drift.
- Existing `audit:*` scanners remain available and forward to compatible lint report families.
- Act and manuscript review prompts include bounded lint evidence when it is available. A lint failure is visible as an advisory; it never becomes a claim that the manuscript passed review.

## Compatibility and boundaries

- Existing projects and scanner commands remain compatible.
- Findings are prompts for editorial judgment, not automatic manuscript edits or release gates.
- The lint does not detect or establish authorship, assess whether text is AI-written, measure literary quality, or replace human editors and readers.
- Install from the pinned `v1.6.0` tag for a reproducible pilot; use a copied or backed-up manuscript for first use.

## Verification

The release candidate runs type, unit, integration, end-to-end, evaluation, deterministic release, packed-extension, and package-dry-run checks. These checks demonstrate software contracts and packaging boundaries, not historical truth, full-book model behavior, literary excellence, publication success, or reader validation.
