# Novel Forge 1.7.0 — Quality orchestration and grounded accuracy

Novel Forge 1.7.0 adds explicit quality tiers, model-spend controls, isolated multi-pass drafting, grounded factual auditing, and an opt-in cost-versus-quality evaluation harness.

## What changed

- **Quality tiers:** economy preserves the existing single host-prompt path; balanced, premium, and editorial add progressively deeper isolated planning, candidate, critic, revision, and verification passes.
- **Budget enforcement:** token and call reservations occur before inference, settle against actual usage when available, and stop or downgrade deterministically when configured ceilings are reached.
- **Isolated workers:** higher-quality passes use Pi print mode without tools, skills, extensions, sessions, or context files. Prompt and manuscript evidence travel through stdin rather than process arguments.
- **Grounded research:** high-risk ready claims require one direct evidence anchor or two independent corroborating anchors. Stored anchors contain source IDs, locators, bounded paraphrases, and optional excerpt hashes—not raw source excerpts.
- **Claim auditing:** proposed chapter claims use exact line ranges and normalized line hashes. Unsupported high-risk claims block; eligible medium/low findings receive at most one targeted qualification or generalization, followed by re-extraction and re-audit.
- **Persistent quality runs:** premium drafting can pause, resume, stop at human gates, and retain budget state without bypassing the canonical guarded event.
- **Telemetry:** schema-two reports store hashes, usage, cost, tier, pass, and safe finish categories. They do not store prompts, prose, model outputs, source excerpts, private reasoning, or credentials.
- **Evaluation:** `npm run eval:quality` creates matched, blinded samples and human review kits only after explicit paid opt-in. Normal CI never runs paid evaluation.

## Authority and compatibility

Existing projects without quality configuration continue to resolve to economy behavior. Intermediate scene plans, candidates, critiques, claim maps, and rejected outputs remain non-canonical cache artifacts. Every accepted creative change still ends in one validated `applyNovelEvent` transaction with stage/hash checks, allowlists, schemas, references, rollback, status/handoff generation, and a Git checkpoint.

Automated diagnostics are not human reader evidence and never update `reader-experiments.yaml`. A green release check does not prove literary quality, factual completeness, publication success, or real-reader validation.

```bash
npm ci

npm test
npm run eval
npm run benchmark:constrained-runtime
npm run benchmark:prompts
npm run verify:release

```
